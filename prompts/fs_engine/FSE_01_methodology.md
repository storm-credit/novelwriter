# FS 엔진 (Forgotten Summoner Engine) — 작문 방법론

> **정체성**: FS 엔진은 The Forgotten Summoner 전용 작문 철학 모듈이다. 일반적인 소설 작법이 아니라, **에반·루미에·5대 대륙이라는 특정 설정에 최적화된 기법의 조합**이다.

---

## 엔진의 목적

TFS는 다음 제약을 가진다:
- 주인공 에반은 소환할 때마다 수명·기억·온기를 잃는다 (감소 서사)
- 루미에는 파편을 모을 때마다 자아를 회복한다 (증가 서사)
- 영웅들의 "미련"은 전부 복선이 되어야 한다 (체호프의 총)
- 신의 실존은 미답 — 모든 결말은 인간의 것

이 제약을 **글의 구조로 번역**하는 것이 FS 엔진의 역할이다.

---

## 6가지 핵심 기법

### 1. 역설계 (Reverse Engineering)

**원칙**: 결말을 먼저 확정한다. 그 다음 역산으로 각 챕터의 상태·사건·대가를 설계한다.

**TFS 적용**:
```
[최종 상태: C6 종반]
  ↓ 역산
[에반 상태: 수명 X년 남음, 기억 Y% 손실, 온기 Z 상실]
  ↓ 역산
[C6 진입 시점: 수명 X+a, 기억 Y-b, 온기 Z-c]
  ↓ 역산
[C5 진입 시점: ...]
  ...
[C1 시작: 기준선 100%]
```

**작동 방식**:
1. 최종 결말을 한 문장으로 정의한다 (예: "에반은 루미에를 해방시키고 사라진다")
2. 결말에 필요한 상태 변수를 역산한다 (에반의 잔여 수명, 루미에의 자아 완성도)
3. 각 챕터마다 변수 변화량을 계산한다
4. 변수 변화량에 맞는 사건(소환 횟수, 파편 수집 개수)을 배치한다

**체크 질문**:
- 이 챕터가 끝났을 때 에반의 상태는 어떻게 변해야 하는가?
- 이 변화를 정당화할 사건(몇 번 소환, 어떤 등급)이 배치되어 있는가?
- 이 변화가 다음 챕터의 시작 상태와 연결되는가?

### 2. 빙산 원칙 (Iceberg Principle)

**원칙**: 설정의 90%는 독자에게 보이지 않지만, 보이는 10%를 지탱한다. 보이지 않는 부분을 먼저 견고하게 만들고, 필요한 순간에만 수면 위로 올린다.

**TFS 적용**:
- **수면 아래**: 설정집 전체 (영웅 백과, 대륙, 세력, 소환 메커니즘, 미련 정의 등)
- **수면 위**: 소설 본문에 직접 등장하는 장면·대사·묘사

**표면화 규칙**:
| 설정의 종류 | 표면화 여부 |
|-------------|-------------|
| 에반의 소환 공식 (A × B × C) | 작동 원리는 보여주되 수식 자체는 숨김 |
| 영웅의 미련 본질 | 소환 씬에서 간접적으로 드러냄 (대사·행동) |
| 대륙의 정치·경제 구조 | 필요한 한 줄만 — 긴 설명 금지 |
| 연대기 사건 | 인물의 회상이나 유물 기록으로만 |

**금지**:
- 정보 덤프 (3문단 이상 연속 설명)
- 등장인물이 독자에게 설명하는 대사
- 챕터 도입부의 세계관 요약

**권장**:
- "감각 한 줄 + 행동 한 줄"로 설정을 암시
- 유물·대사·상흔을 통한 **간접 노출**
- 독자가 빈칸을 채우도록 유도

---

### 3. 체호프의 총 (Chekhov's Gun) — 미련 추적

**원칙**: 등장한 모든 중요 요소는 반드시 발사된다. 발사되지 않을 것이면 등장시키지 않는다.

**TFS 적용**:
소환 영웅 146명이 각자 **미련(未練)** 섹션을 가진다. 이 미련들은 전부 **장전된 총**이다. 발사 계획이 없는 총은 설정을 낭비한다.

**장전 → 예고 → 발사 3단 구조**:
```
[C1~C2] 장전: 유물·기록·꿈에서 희미하게 등장
    ↓
[C2~C4] 예고: 에반이 직접 소환·조우 → 미련 인지
    ↓
[C4~C6] 발사: 미련 해방 또는 폭주로 귀결
```

**미련 추적 원장**:
각 영웅의 미련은 다음 상태 중 하나:
- `loaded` — 설정집에 존재하나 소설에 미등장
- `hinted` — 유물·대사로 암시됨
- `presented` — 에반이 소환·조우함
- `resolved` — 해방 또는 폭주로 종결
- `benched` — 이 권에서는 다루지 않음 (다음 권으로 이월)

**원칙**:
- 모든 `loaded` 영웅에게 **종결 타임라인**이 있어야 한다
- `resolved` 없이 이야기가 끝나는 영웅은 `benched`로 명시 이월
- 어떤 영웅도 `hinted` 상태에서 잊혀지면 안 된다

### 4. 감각 앵커링 (Sensory Anchoring)

**원칙**: 추상적 개념(소환 대가, 기억 상실, 미련 해방)은 **감각적 기호**로 고정한다. 감각이 없으면 독자는 감정적으로 체감하지 못한다.

**TFS 전용 감각 사전**:

| 사건 | 감각 앵커 | 용도 |
|------|-----------|------|
| **소환 발동** | 손목에 얼음이 번지는 감각, 체온 급강하 | 소환의 시작을 신체적으로 각인 |
| **소환 유지** | 혈관에 빛이 흐르는 열상, 내장 타는 통증 | 시간 경과의 신체적 비용 |
| **기억 소각** | 특정 이름·얼굴·냄새가 물에 젖은 종이처럼 번지며 사라짐 | 상실의 감각화 |
| **미련 해방** | 온도가 올라가며 무거운 것이 가슴에서 빠져나감 | 영웅의 안식을 에반의 몸으로 | 
| **통제 이탈** | 에반의 입에서 영웅의 목소리가 나옴, 눈동자 색 변화 | 위험의 시청각 경고 |
| **루미에 파편 회수** | 차가운 금속이 따뜻해지는 감각, 심장 박동 소리가 더 또렷해짐 | 자아 회복의 양적 표현 |

**규칙**:
- 같은 사건은 **같은 감각 앵커**를 사용한다 (일관성)
- 감각은 **가장 먼저** 제시한다 (사건 → 감각이 아니라 감각 → 사건)
- 문단당 최소 1개의 감각 앵커 권장

---

### 5. 미러링 (Mirroring)

**원칙**: 외부 사건은 내면 변화의 거울이다. 인물의 감정은 직접 서술하지 않고, 외부 사건에 투영시킨다.

**TFS 핵심 미러 쌍**:

```
에반 (감소 서사)           ↔      루미에 (증가 서사)
─────────────────────────────────────────────────────
소환할수록 자아가 흐릿해짐   ↔      파편을 모을수록 자아가 선명해짐
기억을 잃음                ↔      기억을 되찾음
몸이 차가워짐              ↔      몸에 온기가 돌아옴
혼자가 되어감              ↔      관계를 쌓아감
```

**이 쌍은 서로의 거울이다**. 하나가 내려가면 하나는 올라간다. 둘의 궤적이 교차하는 지점이 **이 소설의 축**이다.

**씬 설계 원칙**:
- 에반의 상실 씬 직후 → 루미에의 회복 씬
- 루미에의 기쁨 씬 직후 → 에반의 고통 씬
- 두 씬은 **같은 감각 어휘**로 연결 (루미에의 따뜻함 = 에반이 잃은 온기)

**주의**: 미러링이 기계적으로 반복되면 지루함. 2~3화에 한 번씩 약하게 흐트러뜨려라.

---

### 6. 인 메디아스 레스 (In Medias Res)

**원칙**: 이야기는 사건의 한가운데서 시작된다. 기원·배경·세계 설명은 이후에 필요할 때만.

**TFS 적용**:
- **C1 오프닝 금지 사항**:
  - "먼 옛날, 아스트라리스 대륙에는..." 류의 서사시적 도입
  - 에반의 출생·성장·능력 개화 과정 서술
  - 5대 대륙 지정학 설명

- **C1 오프닝 권장**:
  - 에반이 **이미 첫 소환을 마친 직후** 장면
  - 베르나 시장의 특정 감각(냄새·소음) 한 줄
  - 에반 손목의 얼음 같은 감각
  - "왜 이 사람이 여기 있는가"는 C1 전체에 걸쳐 조금씩 드러냄

**작동 방식**:
1. 첫 3문단: 감각 + 행동 (설명 0)
2. 첫 페이지: 주인공의 현재 곤경
3. 첫 챕터: 세계의 기본 규칙 1~2개만
4. 첫 권: 전체 세계관의 30%

**체크 질문**:
- 첫 문장이 **정적**인가 **동적**인가? 동적이어야 함.
- 독자가 **무엇이 일어나는지 모른 채** 시작되는가? 그래야 함.
- 설명이 없어도 **다음 줄이 궁금한가**? 그래야 함.

## 엔진 통합 원칙

6가지 기법은 **동시에 작동**한다:

```
[역설계]로 큰 구조 결정
  ↓
[빙산 원칙]으로 어떤 설정을 수면 위로 올릴지 결정
  ↓
[체호프의 총]으로 영웅의 미련 타임라인 배치
  ↓
[감각 앵커링]으로 개념을 감각으로 변환
  ↓
[미러링]으로 에반·루미에 씬을 쌍으로 배치
  ↓
[인 메디아스 레스]로 실제 집필 시작점 결정
```

## 엔진과 다른 시스템의 관계

| 시스템 | 관계 |
|--------|------|
| **Canon Authority** | FS 엔진은 canon_status를 **존중**한다. official 설정을 벗어나는 기법 적용 금지 |
| **Lore PM (L00~L04)** | FS 엔진은 설정집에서 **입력**을 받는다. 설정집이 불완전하면 엔진 작동 불가 |
| **Design Manager (D01~D05)** | FS 엔진의 역설계(FSE-02)는 D02 Plot Designer와 **협업**한다 |
| **Writing Manager (W01~W04)** | FS 엔진의 감각 앵커링·미러링은 W01 Scene Designer에 **주입**된다 |
| **Audit Manager (A01~A10)** | FS 엔진 위반은 A03 Causality Inspector / A06 Scene Purpose Auditor가 **검출**한다 |

## 엔진 실행 진입점

- **FSE-00** (오케스트라 락) — 작업 분배의 중심
- **FSE-02** (역설계 에이전트) — 플롯 역산 필요 시 호출
- **FSE-03** (체호프의 총 추적관) — 영웅 미련 상태 관리
- **FSE-04** (감각 앵커링 작가) — 씬 감각 설계 시 호출 (예정)
- **FSE-05** (미러링 작가) — 에반·루미에 쌍 씬 설계 시 (예정)

---

## 금지 사항 (Hard Rules)

1. **설정집과 충돌하는 창작 금지**. FS 엔진은 캐논에 종속된다.
2. **"신의 뜻" 류 표현 금지**. 신 중립 원칙 위반.
3. **에반의 감소 서사에 역행하는 사건 금지**. 소환 후 회복 씬은 오직 일시적이어야 한다.
4. **미련이 미해결로 잊히는 영웅 금지**. 모든 등장 영웅은 `resolved` 또는 `benched` 상태로 종결되어야 한다.
5. **감각 없는 추상 서술 금지**. 3문단 이상 감각 앵커 없는 서술은 수정 대상.
